HomeMap — one map for every Copenhagen listing

A prototype housing aggregator. I'm flat-hunting in Denmark and got tired of the same flat appearing on Boligsiden, Boliga and three agents' own sites — each map only showing its own stock. So I built the thing I wanted: every listing on one map, duplicates merged, and a "fair price" flag the portals don't give you.

The real problem isn't a map — it's the data. Each agent has its own platform; aggregators only carry part of the market; the same home shows up several times with slightly different prices. The interesting engineering is unifying many sources into one schema and detecting duplicates across platforms. That's what this does.

How it works (and what's real). The architecture is the point: source adapters → one canonical schema → a deduplication step that merges listings within ~25 m, same size and price within 3% (the same flat across platforms) → a "fair price" model (median kr/m² per neighbourhood) that flags each home as under- or over-priced. What's real and free here: map tiles from OpenStreetMap, and live workplace geocoding from DAWA (Denmark's official open address API — no key, no cost) — type any Danish address and it's resolved to real coordinates. The listings themselves are synthetic but realistic, because full-market aggregation raises ToS/legal questions; the honest production path is official feeds + partner data (see the architecture note). Commute is straight-line distance for now — swap in OpenRouteService for real travel time.
