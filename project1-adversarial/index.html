What's under the hood. Each model is a small convolutional net (two 3×3 conv layers, stride 2 → a dense head) with hand-written backprop — no ML library. FGSM takes one step along sign(∇ₓ loss); PGD iterates that, projecting back into an ε-sized L∞ ball each step — a stronger attack that finds different adversarial examples. Robust training learns on a mix of clean and adversarially-perturbed digits (FGSM, ε=0.12) each step, so its boundary sits farther from real digits without sacrificing too much clean accuracy — the classic robustness–accuracy balance.

The honest caveat: this is a small CNN on MNIST for speed and to keep it 100% client-side. The real flagship version would ship a deeper CNN and a RobustBench model on CIFAR-10 — but the phenomenon you see here (imperceptible perturbation, confident wrong answer, robust model resisting) is exactly the same one that matters in production. Certifying that resistance rigorously leads straight to the SDP relaxations in Project 3.
